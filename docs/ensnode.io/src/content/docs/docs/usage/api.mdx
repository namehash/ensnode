### GraphQL Examples

Fetch data about the three most recently-created domains.

```gql
{
  domains(orderBy: createdAt, orderDirection: desc, first: 3) {
    name
    expiryDate
  }
}
```

## Config API

The Config API provides a complete view of your ENSNode instance's public
configuration, making it easy to verify your setup and understand
how your service is configured.

**Endpoint:** `GET /api/config`

This endpoint returns the public configuration object containing all
the essential information about your ENSNode instance,
including ENS namespace, indexed chains, enabled features, active plugin,
and dependency versions.

### Key Configuration Details

The response includes several important configuration categories:

- **ENS**: current ENS namespace 
- **Network Configuration**: URLs for ENSAdmin interface, public ENSNode API,
  and ENSRainbow service
- **Chains**: Lists of indexed chains
- **Feature Flags**: Boolean settings for experimental features and optional
  functionality
- **Plugins**: Currently active plugins
- **Dependency Information**: Node.js version, Ponder framework version, and
  ENSRainbow schema details

### Example response

```json
{
  "ensAdminUrl": "https://admin.ensnode.io/",
  "ensNodePublicUrl": "http://localhost:42069/",
  "ensRainbowUrl": "http://localhost:3223/",
  "indexedChainIds": [
    1,
    8453
  ],
  "databaseSchemaName": "public",
  "experimentalResolution": false,
  "healReverseAddresses": true,
  "indexAdditionalResolverRecords": true,
  "isSubgraphCompatible": false,
  "namespace": "mainnet",
  "plugins": [
    "subgraph",
    "basenames"
  ],
  "dependencyInfo": {
    "nodejs": "22.17.0",
    "ponder": "0.11.43",
    "ensRainbow": "0.31.0",
    "ensRainbowSchema": 2
  }
}
```

## Indexing Status API

Monitor your ENSNode's indexing progress across multiple chains with
the Indexing Status API. This endpoint provides comprehensive information
about which chains are being indexed, their current status,
and overall indexing status.

**Endpoint:** `GET /api/indexing-status`

Perfect for building monitoring dashboards, health checks, or determining
whether your ENSNode has caught up with recent on-chain activity.

### Understanding the Response

The status response provides both overall and chain-specific information.

### Realtime Indexing Distance

When monitoring uptime or other time-sensitive operations, you may need to
verify that an ENSNode instance has indexed sufficient blocks to maintain
an acceptable distance from the current chain "tip".

The Indexing Status API supports an optional `maxRealtimeDistance` query
parameter that enables this verification.

#### How it works

The `maxRealtimeDistance` parameter accepts a time value in seconds,
representing your application's maximum acceptable lag behind the chain "tip".
When you include this parameter:

- **Success (200 OK)**: The ENSNode instance has achieved the requested
indexing distance
- **Service Unavailable (503)**: The instance hasn't yet indexed enough blocks
to meet your distance requirement

This allows your client to programmatically determine whether the ENSNode
instance is sufficiently synchronized for your use case before proceeding with
operations that depend on recent blockchain data.
