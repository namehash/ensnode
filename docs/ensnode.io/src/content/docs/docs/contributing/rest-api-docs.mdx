The [REST API Docs](https://docs.ensnode.io/ensapi) are generated by Mintlify from an OpenAPI spec. The source of that spec depends on the deployment context. This means production docs always reflect the version of the API running in production, while PR previews document upcoming API changes before they're deployed to production.

### Production Deployments

On each commit to the `main` branch, Mintlify automatically deploys a build of the production docs as follows:

| Content Type       | Source                                                                                                                                            | Output                                                     |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| **REST API Docs**  | OpenAPI spec fetched from `https://api.alpha.ensnode.io/openapi.json` at build time (published by the ENSApi service running in production) | [Production REST API Docs](https://docs.ensnode.io/ensapi) |
| **All other docs** | Committed files on `main`                                                                                                                         | Deploys immediately on merge to `main`                     |

Note: the production OpenAPI spec is generally different than the committed [`docs/docs.ensnode.io/ensapi-openapi.json`](https://github.com/namehash/ensnode/blob/main/docs/docs.ensnode.io/ensapi-openapi.json) file, since the committed file may contain upcoming changes not yet deployed to the API. Non-API documentation (guides, concepts, etc.) may also be published before the corresponding code is released to production.

### Preview Deployments

Mintlify creates a deployment preview for every pull request that changes files in the docs directory. The preview URL is posted as a comment on the PR. In preview deployments, the REST API Docs are generated from the committed OpenAPI spec file rather than the live API, so you can review upcoming API changes before they're deployed.

| Content Type       | Source                                                                                                                                                          | Output                                                           |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| **REST API Docs**  | Committed [`docs/docs.ensnode.io/ensapi-openapi.json`](https://github.com/namehash/ensnode/blob/main/docs/docs.ensnode.io/ensapi-openapi.json) file on the branch associated with the PR | Mintlify preview deployment (link posted as a comment on the PR) |
| **All other docs** | Committed files on the branch associated with the PR                                                                                                                                      | Mintlify preview deployment (link posted as a comment on the PR) |

## OpenAPI Spec Sync Check

On every PR, CI runs an `openapi-sync-check` job that:

1. Regenerates the OpenAPI spec from the ENSApi route definitions by running `pnpm generate:openapi`
2. Compares the freshly generated OpenAPI spec against the committed [`docs/docs.ensnode.io/ensapi-openapi.json`](https://github.com/namehash/ensnode/blob/main/docs/docs.ensnode.io/ensapi-openapi.json), failing with a diff if they don't match
3. Validates that Mintlify can parse the spec with `openapi-check`

If you modify any API route schemas in `apps/ensapi`, you must regenerate and commit the updated spec:

```bash
pnpm generate:openapi
```

Then commit the updated OpenAPI spec to [`docs/docs.ensnode.io/ensapi-openapi.json`](https://github.com/namehash/ensnode/blob/main/docs/docs.ensnode.io/ensapi-openapi.json).
