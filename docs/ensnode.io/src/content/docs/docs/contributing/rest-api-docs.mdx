The [REST API Docs](https://docs.ensnode.io/ensapi) are generated by Mintlify from an OpenAPI spec. The source of that spec depends on context:

| Context         | OpenAPI Source (at Mintlify build time)                              | Output REST API Docs                                            |
| --------------- | ------------------------------------------- | ----------------------------------------------- |
| **Production**  | Fetched from `https://api.alpha.ensnode.io/openapi.json` which represents the OpenAPI spec published by the ENSApi service that's currently running in production *(Note: this is generally different than the committed `ensapi-openapi.json` file on the main branch)* | [REST API Docs](https://docs.ensnode.io/ensapi) |
| **PR Previews** | Committed `ensapi-openapi.json` file on the branch associated with the PR              | Mintlify docs preview page (check each PR's discussion for link)                                    |

This means production docs always reflect the version of the API running in production, while PR previews document upcoming API changes before they're deployed to production.

### Preview Deployments

Mintlify creates a deployment preview for every pull request that changes files in the docs directory. The preview URL is posted as a comment on the PR. In preview deployments, the REST API Docs are generated from the committed `ensapi-openapi.json` file rather than the production API, so you can review upcoming API changes before they're deployed.

| Content Type       | Source                                                   | Behavior                                    |
| ------------------ | -------------------------------------------------------- | ------------------------------------------- |
| **REST API Docs**  | OpenAPI spec fetched from `https://api.alpha.ensnode.io` | Always in sync with deployed production API |
| **All other docs** | Committed files                                          | Deploys immediately on merge to main        |

Non-API documentation (guides, concepts, etc.) may be published before the corresponding code is released to production. The REST API Docs always reflect the actual production API since Mintlify fetches the OpenAPI spec from the production URL at build time.

## OpenAPI Spec Sync Check

On every PR, CI runs an `openapi-sync-check` job that:

1. Regenerates the OpenAPI spec from the ENSApi route definitions by running `pnpm generate:openapi`
2. Compares the freshly generated OpenAPI spec against the committed [`ensapi-openapi.json`](https://github.com/namehash/ensnode/blob/main/docs/docs.ensnode.io/ensapi-openapi.json), failing with a diff if they don't match
3. Validates that Mintlify can parse the spec with `openapi-check`

If you modify any API route schemas in `apps/ensapi`, you must regenerate and commit the updated spec:

```bash
pnpm generate:openapi
```

Then commit the updated OpenAPI spec to `docs/docs.ensnode.io/ensapi-openapi.json`.
