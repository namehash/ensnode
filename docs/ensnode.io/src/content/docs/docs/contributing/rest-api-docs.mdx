The REST API docs are generated by Mintlify from an OpenAPI spec. The source of that OpenAPI spec depends on the context of a doc build. This means the [production REST API docs](https://docs.ensnode.io/ensapi) always reflect the version of the ENSApi instance actively running in production (often different than what's currently committed to the main branch), while preview REST API docs reflect upcoming REST API changes as committed to code in a PR (independent of any deployment of ENSApi).

### Production Doc Builds

On each commit to the `main` branch, Mintlify automatically deploys a build of the production docs as follows:

| Content            | Source                                                                                                                                                                                                                                                                                               | Output                                                     |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| **REST API Docs**  | Mintlify fetches [the version of the OpenAPI spec running in "production"](https://api.alpha.ensnode.io/openapi.json) (Note: often different than [the OpenAPI spec that is committed to the `main` branch](https://github.com/namehash/ensnode/blob/main/docs/docs.ensnode.io/ensapi-openapi.json)) | [Production REST API docs](https://docs.ensnode.io/ensapi) |
| **All other docs** | Committed files on `main`                                                                                                                                                                                                                                                                            | [All other production docs](https://docs.ensnode.io)       |

### Preview Doc Builds

#### Local Preview

You can preview REST API doc changes locally before pushing to a PR. First, regenerate the OpenAPI spec if you've modified any API route schemas:

```bash
pnpm generate:openapi
```

Then start the Mintlify docs site locally:

```bash
cd docs/docs.ensnode.io
pnpm mint dev
```

This starts a local server at `http://localhost:3000` where you can review the REST API docs with your changes.

#### PR Preview

On each push to a pull request branch that changes files in the [`docs/docs.ensnode.io`](https://github.com/namehash/ensnode/tree/main/docs/docs.ensnode.io) directory, Mintlify automatically deploys a "preview" build of the docs as follows:

| Content            | Source                                                                                             | Output                                                                         |
| ------------------ | -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| **REST API Docs**  | The committed `docs/docs.ensnode.io/ensapi-openapi.json` file on the branch associated with the PR | Mintlify preview deployment (Mintlify will post a link as a comment on the PR) |
| **All other docs** | Committed files on the branch associated with the PR                                               | Mintlify preview deployment (link posted as a comment on the PR)               |

## OpenAPI Spec Sync Check

On every PR, CI runs an `openapi-sync-check` job that:

1. Regenerates the OpenAPI spec from the ENSApi route definitions by running `pnpm generate:openapi`
2. Compares the freshly generated OpenAPI spec against the committed [`docs/docs.ensnode.io/ensapi-openapi.json`](https://github.com/namehash/ensnode/blob/main/docs/docs.ensnode.io/ensapi-openapi.json), failing with a diff if they don't match
3. Validates that Mintlify can parse the spec with `openapi-check`

If you modify any API route schemas in `apps/ensapi`, you must regenerate and commit the updated spec:

```bash
pnpm generate:openapi
```

Then commit the updated OpenAPI spec to [`docs/docs.ensnode.io/ensapi-openapi.json`](https://github.com/namehash/ensnode/blob/main/docs/docs.ensnode.io/ensapi-openapi.json).
