The **Omnigraph cookbook** is a set of GraphQL operations aligned with the example queries shipped in [`@ensnode/ensnode-sdk`](https://www.npmjs.com/package/@ensnode/ensnode-sdk) (`omnigraph-api/example-queries`). Each recipe page is a single operation you can copy into your own client, with **variables** resolved for the Sepolia v2 namespace used in these docs.

ENSNode exposes the Omnigraph at `POST {yourEnsNodeBaseUrl}/api/omnigraph` with a JSON body `{ "query": "...", "variables": { ... } }` — the same wire format the TypeScript SDK uses.

## ENSAdmin playground

Each recipe includes a **Run in ENSAdmin** control that opens the hosted [ENSAdmin Omnigraph playground](https://admin.ensnode.io/api/omnigraph) with the query, variables, and connection URL pre-filled so you can execute it in the browser against live indexed data. Use it to explore the schema, tweak selections, and validate responses before wiring the call into an app.

## Example response and cURL

Inside the embedded panel, open the **Response** tab for JSON returned from the public Sepolia v2 ENSNode used in these docs (regenerate with `pnpm -F @docs/ensnode cookbook:refresh-responses` when example queries or variables change). Open **Curl** for a copy-paste `curl` command that posts the same `{ query, variables }` body to the Omnigraph endpoint documented above.
