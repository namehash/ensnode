ENSRainbow is split into two Docker images to minimize long-running rebuilds of the data directory that embeds the rainbow tables for lookups. The `namehash/ensnode/ensrainbow-data` image hosts the computed directory at `/app/data` and `namehash/ensnode/ensrainbow` imports that directory after building the `apps/ensrainbow` project.

## ENSRainbow-data (`namehash/ensnode/ensrainbow-data`)

This image downloads the source `ens_names.sql.gz` and ingests them and produces the output `/app/data` directory.

```bash
# from the monorepo root
docker build -f apps/ensrainbow/Dockerfile.data -t namehash/ensnode/ensrainbow-data .
```

## ENSRainbow (`namehash/ensnode/ensrainbow`)

```bash
# from the monorepo root
docker build -f apps/ensrainbow/Dockerfile -t namehash/ensnode/ensrainbow .
```
